## 1. 项目是什么 / 想验证什么

这是一组基于 **Flyme Auto(魅族车机)** 真实设计语言的网页原型,核心是**重新设计语音交互区域**:

- **旧设计**:语音小窗是一个有框线的固定小窗口(在地图卡左侧)。
- **新设计(本项目要验证的)**:语音区**没有框线**,**宽度固定(~392–424px)**,但**高度随内容动态生长**;当它出现内容时,**把右侧地图卡向右推开**让位。

目标是用这个"会长高的无边框语音区"承载各种丰富度差异很大的语音场景(查天气、导航、调氛围灯、看比赛、深度思考、闲聊、一键舒适、多人游戏),验证这套交互在不同内容密度下都站得住。

**关键约束(贯穿全项目):内容只能显示在语音区那一块** —— 宽度锁死,高度可向下生长,不允许占用语音区以外的屏幕空间(地图卡只是被推开,不是被覆盖)。

## 2. 设计语言 / 核心理念(所有页面共享)

这套"语音区设计语言"是整个项目的骨架,新增任何场景都应遵守:

| 元素 | 做法 |
| --- | --- |
| **车机底座比例** | 1300×812 逻辑舞台(车机 16:10 风格),`fit()` 用 `transform: scale` 等比缩放并居中,黑边留白。 |
| **语音区** | 绝对定位 `left:40; top:64`,宽度由 CSS 变量 `--vz-w`(392~424px)锁定,`flex column` 向下生长,**无边框**(靠毛玻璃卡片自身区分层次)。 |
| **头像 → Aicy 小球** | 左上角头像在唤醒时**形变成 Aicy 语音小球**:`.avatar.awake` → `::before` 旋转锥形渐变光环 + `::after` 白色核心 + 呼吸动画。 |
| **顶栏语音** | 识别结果和回复都放在**顶部状态栏**(不占语音区):`.sv-asr`(逐字滚动识别,完成转绿 `.done`,再淡出 `.gone`)+ `.sv-reply`(TTS 回复文案)。 |
| **地图卡推开** | `.stage.voice-on .map { transform: translateX(calc(var(--vz-w) + 16px)) }`,**在"即将出内容/回复"时**触发(不要在刚识别时就推,太早)。 |
| **内容上沿对齐** | 语音区内容顶部 ≈ 被推开的地图卡上沿(都从 `top:64` 起)。 |
| **思考胶囊** | 可选的 `.think` 折叠药丸 + 可展开 `.think-panel` 步骤列表;记忆步骤 `.is-mem` 用金色 ✦ 高亮 + `memPulse()` 给头像加**仪式感金色光环** `.avatar.mem .mem-halo`。 |
| **配色** | 深色车机桌面为主;卡片 `rgba(22,28,40,.66)` + `backdrop-filter: blur` 毛玻璃;强调色金 `#ffcf6a` / 绿 `#34d06a` / 蓝 `#3b78ff` / 红 `#ff6b6b`。 |

## 5. 技术架构

- **技术栈**:纯原生 HTML/CSS/JS,**无构建、无依赖、单文件**。
- **舞台缩放**:`fit()` = `scale(min(innerWidth/1300, innerHeight/812) * 0.985)`,`resize` 时重算。
- **时间线编排**:`timers` 数组 + `at(ms, fn) => timers.push(setTimeout(...))` 串联动效;`reset()` 清空所有 timer 并复位 DOM(含清除内联 `display:none`、滚动位置、class)。重放靠右上角按钮调 `play()`。
- **地图推开**:给 `.stage` 加/去 `voice-on` 类即可,过渡 `transform .42s cubic-bezier(.2,.7,.2,1)`。
- **动态测高推图**:`syncMapPush()` 实测语音区内容高度,把地图卡向下/右推到不遮挡(闲聊长文本展开时用)。
- **暗色高德风地图**:内联 SVG(道路 casing+core 双层描边、水系、公园、路名、高速盾牌);镜头=对 `.world` 做 transform;图钉是屏幕层 div,由 `applyCam()`/`fitCam()` 重新定位。
- **canvas 特效**:`makeRain()` 下雨;`makeWet()` 采样 `flyme-hero.jpg` 做翻转透镜折射的湿玻璃;`@property --amb` 做氛围色平滑过渡。
- **可复用片段**(跨页面复用,新增场景可直接抄):
  - 顶栏语音(orb + `.sv-asr` + `.sv-reply`)
  - 暗色地图底座 SVG(氛围灯/世界杯/一键舒适/飞花令同款)
  - 思考胶囊 CSS/JS(`.think` + `.think-panel` + 记忆高亮 + `memPulse()`)
  - 地图推开 `voice-on` 机制

```bash
# 浏览器打开 http://localhost:8139/ 选对应 html
```
`.claude/launch.json` 已配置一个名为 `demo` 的 python http server(供预览工具复用)。

# 第二阶段 · 奔驰 MVA「AIOS 动态窗口」(2026-06)

> 这是**当前最新状态**。在第一阶段「无边框语音区」设计语言上,演进成奔驰 MVA 的 **AIOS 动态窗口**:左上 **MVA avatar** + 中部**动态窗口(三视图)** + 底部 **Smart Bar(灵动岛 / 音乐岛 / 电话球)**。20 个 `奔驰 语音区 · *.html` 共享同一套**全局注入**的灵动岛系统。

## A. 全局注入系统(单一事实源)

- **`_inject_island.py`** —— 所有全局「灵动岛 / feed / 组件库 / 音乐岛 / 电话球 / 三视图」逻辑的**唯一来源**。它把一段 IIFE 注入每个页面(18 页 `#mvaOrb` 走替换、2 页 `#fabStream` 走 append)。
- 每页一个 **`*.bak_island`** 备份(注入前的原始页);页面专属改动(如负一屏改造)写在它的 `.bak_island` 里。
- **重新注入工作流(改任何全局行为后都要跑):**
  ```bash
  cd "新文件夹路径/new avatar demo"
  for b in *.bak_island; do cp "$b" "${b%.bak_island}"; done && python3 _inject_island.py *.html
  ```
- 注入的 IIFE 顺序很关键:orb/stage/dock → `var vz`(必须早,音乐岛 playerwrap 在 feed 段之前就要用)→ CSS 注入 → 胶囊 di → 音乐岛 + 布局 → feed 段(feedwrap/组件库/addRec/setView)→ runDemo → busy/settle/fireDemo → 末尾 NEBULA_FEED_DEFAULT / islandshow 钩子。

## B. 本阶段功能清单

**灵动岛(底部 AI):**
- 收起态=光球,展开态=胶囊;两视图切换 **内容态 ↔ 推荐流 feed**。点光球开 feed,再点切回。
- **软暗示**:光球变色(推荐入口=琥珀核 + 脉动光环,内容入口=青绿核)+ 文案变化,不用硬按钮。
- **随机推送**:多数时候只是光球;有推荐才偶尔展开。
- **AI 推荐到达**:feed 在前台→顶部动效插一条;feed 隐藏→灵动岛胶囊提示,且该条已进 feed。

**推荐流 feed + 组件库:**
- 沿用既有 fcard 样式;到高度上限**内部滚动**(不顶到音乐岛);无「为你推荐」标题。
- 每条推荐右侧 **✕ 删除按钮**(替代原 › 箭头)。
- **右滑 feed → 组件库**(iOS 负一屏风格真实预览图廊);顶部固定区**最多 1 大 2 小**;固定区=**蓝色玻璃磁贴 + 左侧高亮蓝条**,与推荐行之间有**分割线**。

**音乐岛:**
- 常驻;有歌=胶囊(封面 + 跑马灯标题 + 控制),无歌=球;**通话中强制收球**。
- 暂停 UX:暂停后**保持展开 2 秒**再收起,2 秒内再播则继续。
- 点胶囊主体 → **音乐小窗**(第三视图,全局)。

**电话球 / 双球:**
- 通话态身份球;音乐/电话页**双球并排**(AI 球 + 身份球,身份球在灵动岛**右侧竖直对齐、不被遮挡**,用 stageX/stageY 布局坐标计算)。

**负一屏页(车控媒体+Feed):**
- 整页换成全局灵动岛系统:**默认进 feed 视图**;内容视图保留 MVA 记忆/媒体/续航;删掉原页自己的金色 pin 区 + 语音固定。
- **首次进入**:浮在 feed 上方的「**右滑添加组件**」手势指引,播两次后自然消失。

**复杂导航页(导航规划与音乐卡片切换):**
- 思考胶囊在**扎点完成后消失**。
- 开始导航 = TBT 转向卡 + **右侧竖向点位进度轨**(走到哪亮哪,绿色连线填充)。
- 导航中**「不去接孩子了」重规划**:思考态回归(车继续走、TBT 下移腾位)→ 完成后思考消失、TBT 复位、进度轨去掉接孩子。

**回滚的探索:**
- ASR**「收音光效」**(声纹条 / 流光胶囊 / 全屏边缘流光)探索后**已回滚**;ASR 仍按「识别文字上屏」。当前文档/产品不含光效。

## D. 工作流 / 工具

- **重新注入**:见 A 节(改全局行为必跑)。
- **GIF 生成 pipeline**:无头 Chrome(`puppeteer-core`,用系统 Google Chrome,不下 Chromium)逐帧截图 → **Pillow** 编码(按真实时间戳定帧速)。脚本在 `/tmp/gifcap/`(`capture.js` 触发各页主演示并截图、`encode.py` 编码;`?islandshow=1` 专截灵动岛)。**需先起本地 http server**(`python3 -m http.server`),否则截到 ERR_CONNECTION_REFUSED。
